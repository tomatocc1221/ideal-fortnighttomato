# 开发日志 — 2026-05-24

## 完成事项

### Phase 2 核心功能：球队管理系统

- [x] **赛后录入系统** — 管理后台新增「赛后录入」标签页，选择已结束比赛录入比分、进球者、助攻者，自动判定胜负
- [x] **出勤统计** — `/api/stats` 端点计算每位球员正选/候补/请假次数 + 出勤率百分比，后台表格颜色分级展示
- [x] **费用分摊** — 场地费 ÷ 正选人数自动计算人均费用，报名面板实时显示
- [x] **连续缺席提醒** — 连续 ≥2 次请假的球员在后台报名记录中黄色高亮标注
- [x] **比赛 2 小时时长机制** — 比赛开始 2 小时后自动跳过，进入后台等待赛后录入
- [x] **后端服务器** — Node.js `server.js`（端口 3456），REST API 完整 CRUD，JSON 文件数据库
- [x] **报名系统** — 球员姓名+个人密码验证，正选/候补/请假三态，自动补位，请假原因
- [x] **管理后台** — `admin.html`，5 个标签页（队员管理/比赛管理/报名记录/出勤统计/赛后录入）
- [x] **API 封装层** — `js/api.js` 统一管理所有后端请求，失败时静默降级
